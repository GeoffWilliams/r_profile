Overview

R_profile::Linux::Rsyslog

Setup rsyslog client:

  • Install Package
  • General settings
  • Managed logfiles

Examples:

Setup syslog client with settings from hiera

include r_profile::linux::rsyslog

Hiera data for general rsyslog settings

r_profile::linux::rsyslog_client::settings:
  - '$FileOwner': 'root'
  - '$FileGroup': 'root'
  - '$FileCreateMode': '0600'
  - '$DirOwner': 'root'
  - '$DirGroup': 'root'
  - '$DirCreateMode': '0750'

Hiera data for rsyslog entries

r_profile::linux::rsyslog_client::entries:
  'daemon.*': /var/log/daemon.log
  'syslog.*': /var/log/syslog

Parameters:

  • settings (Array[Hash[String, Any]]) (defaults to: []) —

    Settings to insert into the rsyslog.conf file. Existing definitions will be replaced in-place. If a setting is added that doesn't already exist, it will be inserted into the file from line setting_insertion_line. Settings will be processed in array order (see examples)

  • entries (Hash[String,String]) (defaults to: {}) —

    Hash of logging entries to create (see examples)

  • service_ensure (Enum['running','stopped']) (defaults to: "running") —

    How to ensure the rsyslog service

  • service_enable (Boolean) (defaults to: true) —

    true to start rsyslog on boot otherwise false
